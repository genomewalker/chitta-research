chitta-research

An autonomous research operating system. Three agents run a closed loop — generate hypotheses, execute experiments, score results — against a persistent typed belief graph.

What makes this different

Typed belief graph

Eight node types, five epistemic edge kinds. Every hypothesis carries prior and posterior confidence. Serialized to JSON and restored across sessions — nothing is discarded.

6D Pareto fitness

Novelty, empirical gain, reproducibility, cost efficiency, transfer potential, calibration improvement. No single weighting imposed; the Pareto frontier is maintained continuously.

Reconsolidation

When a belief is corrected, confidence reductions cascade immediately to all downstream nodes via DerivedFrom traversal. BlockedBy edges are inserted in the same cycle.

chitta-field native

Agents write memories, triplets, and observations to the chittad daemon over Unix socket. HNSW + BM25 + SDR cortical index stores research events durably.

Multi-model rooms

Run a structured parallel debate across Claude, local Ollama models (gemma4, deepseek-r1), and Codex before producing a synthesized structured answer.

No API key required

The claude-cli provider delegates to claude -p, reusing the authenticated Claude Code session. Local models via Ollama need no credentials at all.

Architecture

agenda.yaml | v cr-agenda ──> BeliefGraph (cr-graph + cr-types) | ┌───────────────┼───────────────┐ v v v Hotr Adhvaryu Udgatr (hypotheses) (executor) (scorer) | | | +───────────────+───────────────+ | AgentContext (cr-agents) | | | | LLM Chitta Artifacts Resources (cr-llm)(cr-chitta)(cr-artifacts)(cr-resources) | chittad daemon (Unix socket) HNSW + BM25 + SDR cortical index Brahman -- portfolio arbiter + cross-domain transfer cr-fitness -- Pareto frontier computation cr-reconsolidation -- downstream confidence propagation

Three Tokio tasks run concurrently, each polling its own step() on a 100 ms loop. Brahman runs in a fourth task evaluating portfolio quality and injecting cross-domain transfer memories. The orchestrator shuts down when all agents are idle or the budget is exhausted.

The four agents

Hotr
Hypothesis generation

Reads the belief graph, finds unanswered questions, sends context to the LLM. Adds Hypothesis and ExperimentPlan nodes with prior confidence.

Adhvaryu
Experiment executor

Finds ExperimentPlans with no Run child. Sends steps to the LLM. Adds Run nodes with status, token accounting, and a git SHA for the committed artifact.

Udgatr
Analyst & scorer

Finds succeeded Runs without fitness scores. Scores on all six fitness dimensions. Updates posterior confidence. Adds Claim nodes for strong results.

Brahman
Portfolio arbiter

Monitors convergence across all participants in a multi-model room. Detects stagnation, scores novelty, stores cross-domain transfer opportunities in chitta memory for future sessions.

Multi-model rooms

Run a structured parallel debate. Each participant can be a different model and backend. All participants respond to the same prompt in parallel; the synthesizer produces the final structured answer.

# agenda.yaml snippet
llm:
  provider: room
  model: "aDNA compression codec selection"
  room:
    rounds: 1
    participants:
      - name: Critic
        backend: claude-cli
        system: "You are a bioinformatics critic..."

      - name: Empiricist
        backend: openai-compat
        model: gemma4:26b
        base_url: http://gpu-node:11434/v1
        system: "You are an empiricist..."

      - name: Theorist
        backend: openai-compat
        model: deepseek-r1:32b
        base_url: http://gpu-node:11434/v1
        system: "You are an information theorist..."

      - name: Engineer
        backend: codex
        model: gpt-5.4
        system: "You are a systems engineer..."

    synthesizer:
      name: Synthesizer
      backend: claude-cli
      system: "You are a JSON synthesizer. Resolve any contradictions.
               Output raw JSON only — no prose, no markdown."
See agenda.test-local.yaml in the repo for a complete working room example covering aDNA FASTQ compression (Claude + gemma4:26b + deepseek-r1:32b + gpt-5.4).

chitta integration

The cr-chitta crate connects to chittad over a Unix socket using the JSON-RPC MCP protocol. Three operations are used:

  • remember — Hotr records hypothesis events; Adhvaryu records experiment outcomes.
  • observe — Reconsolidation writes structured observations for the daemon's subconscious.
  • triplet — Udgatr writes hypothesis:ID confirmed_by run:ID with posterior confidence as weight.
The daemon is optional. If the socket is unavailable, all agent logic continues without it — chitta is an enhancement, not a hard dependency.

Design notes

The three agent names — Hotr, Adhvaryu, Udgatr — are the three primary priests of the Vedic yajna (ritual fire). Hotr recites invocations; Adhvaryu performs the physical ritual actions; Udgatr chants the responses. The research loop mirrors this structure: invocation (hypothesis), execution (experiment), evaluation (scoring).

The fitness vector's six dimensions were chosen to avoid optimizing for novelty alone. Calibration improvement specifically rewards experiments that update the system's confidence in proportion to actual evidence.

chitta-research was designed in a three-model room debate (GPT-5.4 + Gemini-2.5-Pro + Claude Sonnet) to stress-test the architecture against adversarial critique before a line of code was written.
