Research Operating System

Autonomous research
with persistent memory.

chitta-research is a Rust system that runs three concurrent agents in a closed loop — generating hypotheses, executing experiments, scoring results — against a typed belief graph that accumulates structured knowledge across every session. Unlike one-shot autoresearch pipelines, it remembers what it tried, what failed, and what those failures mean for everything downstream.

Why it exists

Systems like autoresearch, AIDE, and AI Scientist treat each run as independent. They produce a report, not a knowledge base. chitta-research is built around the premise that scientific reasoning is inherently cumulative — and that the structure of what you know matters as much as the content.

Typed belief graph

Eight node types connected by five epistemic edge kinds. Every hypothesis carries prior and posterior confidence. Claims record which observations support them. The graph persists across runs via JSON snapshot.

6-dim Pareto fitness

Novelty, empirical gain, reproducibility, cost efficiency, transfer potential, and calibration improvement. No single weighting is imposed. Dominated results are pruned; trade-offs are preserved.

Reconsolidation

Corrections to high-priority beliefs propagate immediately to all downstream nodes via DerivedFrom traversal. BlockedBy edges mark the affected subgraph. Dead ends lower chitta recall scores for future searches.

No API key required

The default provider spawns claude -p, reusing the authenticated Claude Code session. No separate billing. Anthropic API, OpenAI, and Codex exec are also supported.

chitta-field native

Agents write memories, triplets, and observations to chittad over a Unix socket. HNSW + BM25 + SDR cortical indexing. Failed experiments lower domain realm scores so future hypothesis generation avoids proven dead ends.

git-backed artifacts

Every experiment result is committed to a local git repo. Commit SHAs are stored on Run nodes. Full provenance: graph node → git commit → file content. Reproducibility is a first-class citizen.

Architecture

An 11-crate Cargo workspace. Three Tokio tasks run concurrently on a 100 ms polling loop. The orchestrator shuts down when all agents idle out, the budget is exhausted, or SIGINT is received.

The research loop

Each cycle, three agents concurrently poll the shared belief graph (protected by a tokio::sync::RwLock) and take the next available action. Work is naturally serialized by what exists in the graph — Hotr must generate hypotheses before Adhvaryu can execute them, and Adhvaryu must produce runs before Udgatr can score them.

1

Hotr — hypothesis generation

Selects the highest-priority ResearchProgram. Finds its unanswered Questions (no Hypothesis children via DerivedFrom edges). Sends the question and domain context to the LLM. Adds 2–3 Hypothesis nodes and ExperimentPlan children. Records the event in chitta with tags [hotr, hypothesis].

2

Adhvaryu — experiment execution

Finds ExperimentPlan nodes with no Run child. Acquires a resource semaphore slot. Sends the plan steps and hypothesis to the LLM. Commits results.json to the artifact git repo. Adds Run, Observation nodes. Charges cost against the budget. Records the outcome in chitta.

3

Udgatr — scoring and claims

Finds succeeded Runs with no FitnessVector. Sends hypothesis, plan, and observations to the LLM for structured 6-dim scoring. Updates the Run's fitness. Updates hypothesis posterior confidence. For empirical_gain > 0.6, creates Claim nodes with Supports or Refutes edges. Writes a triplet (hypothesis:ID confirmed_by run:ID) to chitta.

4

Shutdown and snapshot

When all three agents are idle for --max-cycles consecutive cycles, or the total budget is exhausted, or SIGINT arrives, the loop terminates. The full belief graph is written to graph_state.json and is available for the next run or external analysis.

Quickstart

Requires Rust 1.92.0 (the rust-toolchain.toml pins the version). Use ./build.sh rather than calling cargo directly on HPC systems — it clears the conda linker override that causes link failures.

1. Build
# Clone and build
git clone https://github.com/genomewalker/chitta-research
cd chitta-research
./build.sh build --release
2. Test the pipeline (no external calls)
# --mock uses a deterministic stub LLM
# Full pipeline runs: graph build, agent steps, artifact commits, scoring
./build.sh run --release --bin chitta-research -- \
    --agenda agenda.example.yaml \
    --mock \
    --max-cycles 5
3. Run for real (claude-cli, no API key)
# Requires `claude` on PATH (Claude Code installation)
./build.sh run --release --bin chitta-research -- \
    --agenda agenda.example.yaml \
    --max-cycles 20
agenda.yaml format
programs:
  - title: "ARG prevalence in ancient permafrost"
    domain: metagenomics
    questions:
      - "Do antibiotic resistance genes increase with permafrost depth?"
      - "Are ancient ARGs phylogenetically distinct from modern variants?"
    methods:
      - "BLASTp against CARD"
      - "Short-read assembly with MEGAHIT"
    priority: 1.0
    max_budget_usd: 5.0

budget:
  total_usd: 10.0     # hard ceiling across all programs
  gpu_slots: 0
  cpu_workers: 4

llm:
  provider: claude-cli  # no API key needed
  model: claude-sonnet-4-6

chitta:
  mind_path: ~/.claude/mind

Crate structure

11 crates in the Cargo workspace. Each has a single responsibility.

cr-types
Core types: NodeKind, EdgeKind, FitnessVector, RunStatus, ResourceUsage. All serde-serializable.
cr-graph
BeliefGraph — petgraph-backed stable directed graph with typed lookup, evidence queries, JSON snapshot I/O.
cr-agents
Agent trait + AgentContext + three implementations: Hotr, Adhvaryu, Udgatr.
cr-fitness
Pareto frontier computation over FitnessVector arrays. Pure function, no I/O.
cr-llm
LlmClient trait + ClaudeCliClient, AnthropicClient, OpenAiClient, CodexClient, MockLlmClient.
cr-chitta
Async Unix socket client for the chittad daemon. JSON-RPC MCP tools/call protocol.
cr-reconsolidation
Downstream confidence propagation when a belief is corrected. Inserts BlockedBy edges.
cr-artifacts
git2-backed artifact store. Commits result files to a local repo; returns SHA for graph nodes.
cr-resources
Semaphore-based GPU/CPU slot manager + atomic budget accounting.
cr-agenda
YAML parser for the agenda format. Converts programs and questions into the initial BeliefGraph.
cr-daemon
Binary entry point. CLI parsing, context construction, agent task spawning, event loop, graph snapshot.
