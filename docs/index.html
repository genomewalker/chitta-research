What this is

Existing autonomous research systems — autoresearch, AIDE, AI Scientist — share a structural flaw: they forget. Each run starts cold. The agent re-learns what failed last time. The experiment log is either a flat TSV or a git history that requires reading to extract meaning.

chitta-research is built differently. Every hypothesis, experiment, observation, and correction is a first-class node in a typed belief graph that persists across sessions. When an experiment fails, the failure is reasoned about structurally — domain reliability drops, downstream beliefs are reconsolidated, and future hypothesis selection accounts for what was learned. The system gets smarter with each run, not just within each run.

Three agents

Each agent has one job, one access pattern, and one failure mode.

Hotr
hotṛ — the invoker

Hypothesis Generator

Finds Question nodes in the belief graph with no child Hypotheses. Recalls similar past experiments via HNSW semantic search, weighted by PoE domain reliability. Calls the LLM with this grounded context to generate structured hypotheses plus experiment plans. Adds them as typed graph nodes with DerivedFrom edges.

Adhvaryu
adhvaryu — the ritual technician

Experiment Executor

Picks the highest-fitness ExperimentPlan with no child Run. Acquires a resource slot from the semaphore manager. Executes the experiment — either via LLM simulation or a subprocess. Commits raw artifacts to the git store. Creates a Run node with full provenance: plan ID, git commit hash, elapsed time, resource usage.

Udgatr
udgātṛ — the chanter

Analyst

Finds Succeeded Run nodes with no FitnessVector. Calls the LLM to score on six dimensions. Updates posterior confidence on the parent Hypothesis. Creates Claim nodes for strong findings. Adds CONFIRMS or REFUTES edges. When a high-confidence belief is refuted, triggers immediate reconsolidation — not a nightly batch.

Memory

A belief graph that decays, reconsolidates, and tracks domain reliability.

Graph node types

ResearchProgramTop-level container. Owns budget, priority, domain.
QuestionAn open research question within a program.
HypothesisTestable statement. Carries prior and posterior confidence.
ExperimentPlanExecutable protocol. Linked to one Hypothesis.
RunExecution record. Git commit hash, resource usage, outcome.
ObservationRaw measurement from a Run.
ClaimStatistically significant finding. May carry a confidence scope.
MethodReusable protocol extracted across experiments.

Pareto fitness dimensions

  • novelty0.0 – 1.0
  • empirical_gain0.0 – 1.0
  • reproducibility0.0 – 1.0
  • cost_efficiency0.0 – 1.0
  • transfer_potential0.0 – 1.0
  • calibration_improvement0.0 – 1.0

Experiments are ranked by Pareto dominance, not a scalar. A run that scores high on empirical gain but low on reproducibility will not crowd out a more reproducible result. Budget is allocated proportionally across the Pareto frontier.

When chitta is connected, domain reliability (PoE) from the chittad daemon multiplies recall scores at query time — realms with many corrections naturally surface less.

The loop

Six steps, repeating until budget is exhausted or the agenda is answered.

Recall

Hotr queries the belief graph and chitta memory for open Questions and past experiments similar to the current agenda. PoE-weighted recall means directions that have failed before get less attention.

Hypothesize

The LLM generates testable hypotheses grounded in recalled context — not blank-slate generation. Each hypothesis becomes a graph node with a prior confidence and DerivedFrom edges to its source Question.

Execute

Adhvaryu selects the highest-fitness unexecuted plan, acquires a resource slot, runs the experiment, and commits the raw artifacts to git. The Run node records full provenance.

Score

Udgatr computes a 6-dimensional FitnessVector for each completed run and updates the posterior confidence on the parent Hypothesis. Claims are created for statistically significant findings.

Reconsolidate

When a correction refutes a belief with tier ≥ 2, downstream nodes are invalidated immediately — not at the next nightly batch. The graph stays coherent across hundreds of experiments.

Persist

The belief graph snapshot is written to disk. If chitta is connected, key findings are stored as typed memories with triplet edges, becoming retrievable across future sessions and projects.

Get started

Clone, build, define an agenda, run.

1. Build

# requires Rust 1.92+ and cargo git clone https://github.com/genomewalker/chitta-research cd chitta-research ./build.sh build # test the full pipeline without any API key ./build.sh run --bin cr-daemon -- \ --agenda agenda.example.yaml \ --mock --max-cycles 5

The --mock flag uses an internal LLM stub that returns structured responses instantly — runs the complete Hotr→Adhvaryu→Udgatr pipeline without any external calls.

2. Write your agenda

programs: - title: "ARG prevalence in ancient permafrost" domain: metagenomics questions: - "Do ARGs increase with permafrost depth?" - "Are ancient ARGs phylogenetically distinct?" methods: - "BLASTp against CARD database" priority: 1.0 max_budget_usd: 5.0 llm: provider: claude-cli # uses `claude -p`, no key model: claude-sonnet-4-6 chitta: mind_path: ~/.claude/mind # optional

LLM providers

claude-cliUses claude -p via stdin. No API key. Reuses your current Claude Code session auth.
codexUses codex exec non-interactively. Requires codex CLI installed.
anthropicDirect API. Set ANTHROPIC_API_KEY.
openaiDirect API. Set OPENAI_API_KEY.
mockInternal stub. Pass --mock flag. No external calls. For testing.
